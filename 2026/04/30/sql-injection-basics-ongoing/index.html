--注释的应用

注释 : 用注释来截断查询

当一个网站需要用户密码来登录时，SQL会通过查询来验证有没有这个用户，成功则登录

但是可以通过 -- 注释掉WHERE字句中的密码查询部分，来进行管理员登录

从数据库表中检索数据

当程序存在SQL注入漏洞，可以通过 UNION 来检索其他表的数据，并且把结果附加到原始查询

要 UNION 正常工作有两个关键要求：

  • 各个查询返回相同数量的列
  • 各个查询每一列的数据类型需兼容

确定几列
方法1：’ ORDER BY 1-- //这个数字可以从大到小，直到不发生错误
方法2：’ UNION SELECT NULL,NULL-- //空值的数量根据情况添加，直到发生错误

寻找可用的数据类型的列

一般需要的数据都是字符串形式，所以需要找到数据类型是字符串或者字符串数据兼容的，可以通过 UNION SELECT 有效负载来为每列放置字符串值，如果没有报错且返回包括注入字符串在内的额外内容，那就找对了

OAST 外带盲注

当程序使用异步执行，用多条线程来执行任务时，上面的所有注入均无响应。在这种情况下可以利用外带网络交互来注入，且返回数据。（通常使用DNS）

这里需要用到burp suite 专业版的 collaborator

DNS 查询：让数据库对外部域名DNS查询 ，主要查询是否能外带，且存不存在漏洞

DNS 数据外泄：让数据库对包含注入查询结果的外部域名进行DNS查询，返回数据

堆叠查询注入

批量执行多个查询，但只返回第一个结果。一般通过第二次查询触发 DNS 查询、条件错误或时间延迟

存储型SQL 注入

普通的SQL注入主要发生在程序处理HTTP请求的输入，并以不安全的方式把输入整合到SQL查询中。而存储型就是先把用户输入存储起来，等待后面的某一次HTTP请求这个存储的数据，并以不安全的方式整合到SQL查询中。

防止SQL注入

参数化查询：在一般的查询中是把用户的输入直接拼接到SQL语句中，而参数化查询就是先写好一个框架，再把用户的输入用占号符 ‘?’ 来代替。这样做的好处就是把执行指令和输入数据彻底分开。可以用于：WHERE, INSERT, UPDATE 等等
